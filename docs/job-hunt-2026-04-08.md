# Job Hunt Mode — linkedin-cli Integration

> 将 grunk job-hunt pipeline 与 linkedin-cli prospect pipeline 打通，形成完整：
> **职位研究 → 内推找人 → 精准触达** 闭环。

## 使用方式

在 Claude Code 里激活 skill：
```
/linkedin-job-hunt
```

或在任意会话直接用命令：

```bash
# 1. 用 browse 做职位市场研究（无需 LinkedIn 登录）
opencli linkedin search "software engineer Go remote" --limit 20

# 2. 用 search-people 找目标公司员工
opencli linkedin search-people "software engineer at DoorDash Toronto" --limit 10

# 3. 用 prospect pipeline 做候选人评分 → 触达
./scripts/prospect.sh search "software engineer at DoorDash" --limit 20
./scripts/prospect.sh scan      # 评分（A/B/C/D tier）
./scripts/prospect.sh review     # 人工审核
./scripts/prospect.sh outreach --template templates/hco-intro.txt --dry-run
```

## job-hunt Skill 触发条件

- 用户提到 job hunt、职位搜索、投简历、内推
- 用户提到 `career-ops`、`autoresearch`、`grunk job-hunt`
- 需要找人做 referral 时

## 完整 Pipeline 流程

```
[grunk job-hunt]                    [linkedin-cli prospect]
─────────────────                  ────────────────────────
1. 市场研究                          1. 找人
   browse LinkedIn Jobs                search-people <company>
   ↓                                 ↓
2. 评估 Job URL                    2. 评分
   grunk career <url>                 prospect.sh scan
   ↓                                 ↓
3. 找内推                           3. 触达
   search-people <company>             prospect.sh outreach
   ↓
4. 发连接请求附上 referral 请求
```

## 评分标准（Tier A/B/C/D）

- **A** — authority >= 12 且 relevance >= 3（大品牌决策者）
- **B** — authority >= 6 或强行业 + 网络可达
- **C** — 在目标行业，authority 低
- **D** — 未过门控

详见 `scripts/prospect.sh` 的三级门控级联逻辑。
